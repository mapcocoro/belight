## 現状サイトの分析結果

### 技術構成
- WordPress 6.0.11
- Contact Form 7 プラグイン
- カスタムテーマ（BEM風CSS）

### 問題点

#### パフォーマンス・技術面
- インラインCSS約25KB埋め込み → キャッシュ効かない
- WordPress + 複数プラグインで重い構成
- 画像の遅延読み込み未対応

#### デザイン・UI
- タイポグラフィの統一感なし（Times New Romanとゴシック混在）
- CTAボタンが目立たない
- 余白が詰まりすぎ
- **フッターに「ここにキャッチフレーズが入ります」の仮テキスト残存**

#### コンテンツ
- お知らせが「ホームページ作成しました」の1件のみ → 放置感
- 実績一覧20社あるが説明なし
- サービスの差別化ポイントが弱い（数値・具体例なし）
- 信頼性要素なし（お客様の声、認証バッジ等）

#### UX・導線
- ナビゲーションが冗長
- モバイルメニューのサブメニュー発見しにくい
- 問い合わせへの誘導が弱い

#### アクセシビリティ・SEO
- alt属性が不十分
- OGPタグなし
- 構造化データが最低限のみ

## 製品情報（実データ）

### アルミスタンション
- **特徴**: アルミニウムは鋼材の約1/3の軽さ、磁石クレーンに吸着しない、赤錆が発生しない
- **重量貨物用（L=1000）**: 鋼材17kg → アルミ7.7kg
- **重量貨物用（L=1500）**: 鋼材20.5kg → アルミ11.6kg
- **軽量貨物用（L=1000）**: 鋼材9.5kg → アルミ4.6kg
- **軽量貨物用（L=1500）**: 鋼材14.3kg → アルミ6.9kg
- **耐荷重**: 重量貨物用 2t/本、軽量貨物用 1t/本
- **形状**: ストレート、4の字、伸縮ロープ受けスタンション

### アルミパレット
- **特徴**: ゴミが発生しにくい、積み荷に合わせた形状で製作可能
- **メリット**: 食品・自動車部品の現場で木片混入の心配なし、リサイクル性に優れる
- **バリエーション**: 汎用パレット、異形状パレット、八角形パレット、ストレッチャー兼用パレット

### アルミラック
- **特徴**: 合金により鉄と同等の強度、静電気が発生しないため引火の心配なし
- **実績**: 世界初の水素混焼エンジン搭載商業旅客船『ハイドロびんご』向け水素ボンベラック

## 現行サイト vs 提案サイト 比較

### 技術面の比較

| 項目 | 現行（WordPress） | 提案（スクラッチ） |
|------|-------------------|-------------------|
| 表示速度 | 重い（WP+プラグイン） | 高速（静的HTML or Next.js） |
| CSS | インライン25KB（キャッシュ効かない） | 外部CSS（キャッシュ有効） |
| セキュリティ | WP脆弱性リスクあり | 静的なら攻撃面なし |
| 保守 | WP・プラグイン更新必要 | ほぼ不要 |
| サーバー費用 | PHP+DB必要 | 静的ホスティングで安価 |
| CMS | WordPress管理画面 | MicroCMS（社長の独り言用） |

### デザイン・UI面の比較

| 項目 | 現行 | 提案 |
|------|------|------|
| フォント | Times New Roman + ゴシック混在 | 統一（Noto Sans JP等） |
| 余白 | 詰まりすぎ | 適切な余白で読みやすく |
| CTAボタン | 目立たない | 明確なアクセントカラー |
| ファーストビュー | 情報が散漫 | メッセージを絞って訴求 |
| モバイル対応 | サブメニュー使いにくい | タップしやすいナビ |

### 具体的な改善ポイント

| 箇所 | 現行の問題 | 提案での改善 |
|------|-----------|-------------|
| フッター | 「ここにキャッチフレーズが入ります」残存 | 経営理念・連絡先を正しく表示 |
| お知らせ | 「ホームページ作成しました」1件のみ → 放置感 | 社長の独り言で定期更新 → 活きたサイトに |
| 製品ページ | 製品ごとに別ページ、行き来が面倒 | 1ページで概要把握 → 詳細は別ページ |
| 導入実績 | ロゴ20個が並ぶだけ | 同じ内容でもレイアウト改善で見やすく |

### 移行で変わること・変わらないこと

**変わらない（コンテンツはそのまま）:**
- 会社概要の内容
- 製品説明文
- 経営理念
- 代表挨拶
- 連絡先情報
- 導入実績企業リスト

**変わる（技術・デザイン改善）:**
- 見た目・レイアウト
- 表示速度
- 更新のしやすさ（MicroCMS）
- スマホでの操作性
- フォント・配色の統一感

> **提案の要点: 「中身は同じ、器を良くする」**

- ターゲット: 先進的・国際的な印象を与えたい場合

## デモサイトの共有方法

### 方法1: GitHub Pages（推奨・無料）

1. GitHubリポジトリをPublicにする
2. Settings → Pages → Source を「main」ブランチに設定
3. 数分後に `https://[username].github.io/[repo-name]/pattern1/` でアクセス可能

### 方法2: Vercel（本番環境と同じ）

1. Vercelにログイン（https://vercel.com）
2. 「New Project」→ GitHubリポジトリを選択
3. デプロイ後、自動生成されたURLをクライアントに共有
4. 本番と同じ環境で確認できるのがメリット

### 方法3: ローカルファイルを直接送付

- `pattern1/index.html` をそのままメール添付
- クライアントがブラウザで開けば確認可能（ただし一部機能が動作しない場合あり）
